# 受不完美验证器约束的迭代自我提升:自训练何时崩溃,能否预测?

*实验层主张(受控测量)在设计上即确定;分析层主张(相结构、假阳/假阴、可预测律)是数据收集后才评估的假设。*

---

## 摘要

大语言模型越来越多地通过「自己生成训练数据 → 用一个验证器(verifier)筛选 → 在筛过的数据上微调」来自我提升(STaR、ReST^EM、RLAIF、o1/R1 式流程)。在一切现实场景中,验证器都是**不完美**的——它是一个学出来的奖励模型或 LLM 评判器,而非真值预言机。然而,既有工作都没有系统回答这个核心问题:**验证器要多准,迭代自训练才能持续提升而非崩溃,这个边界能否预测?** 我们把验证器准确度当作一个**受控、可连续扫描**的变量,放进标准的迭代「自消费」SFT 循环里,在「两参数验证器(假阳率 ρ₀、假阴率 ρ₁)× 模型规模 × 模型家族 × 任务」上测量自训练轨迹。这一测量——受控验证器准确度 × 迭代 SFT × 跨规模/家族/任务——在文献中尚不存在。我们给出该实验可交付结果的**上限**(一条普适且可预测的相界律)与**下限**(首份系统性经验刻画,可能是异质性/负结果)。

## 1. 引言

自我提升——模型从自生成数据中自举能力——支撑了近期诸多进展(STaR、ReST^EM、Self-Rewarding LMs、RLAIF、推理 RL)。其机制是一个循环:**生成 → 验证/筛选 → 训练 → 重复**。关键在于,模型训练用的是它**自己**的生成,而非外部人写的标准解;唯一的外部锚定就是那个决定「哪些自生成可以拿来学」的验证器。

两个事实让验证器质量成为枢纽变量。第一,**没有**验证器时(或验证器=模型自身似然),递归自训练已知会**崩溃**——错误复利累积、分布收窄(model collapse)。第二,在最要紧的现实场景里——开放式生成、没有可判答案的推理、RLAIF——验证器是一个**学出来的、不完美**的模型,而非真值检查器。因此自我提升的成败,取决于**这个不完美的外部验证器有多好**。尽管如此,这一关系从未被刻画:**验证器差到什么程度,迭代自训练就从"有益"翻转为"有害",这个边界是否随模型规模、家族遵循一条可预测的律?**

我们的贡献清晰地分为两层:

- **实验层(novelty 与结论无关)**:我们产出首份系统性测量——在**受控、可扫的外部验证器准确度**下、跨模型规模(Qwen2.5 0.5B–14B)、六个家族、两个任务的迭代自消费 SFT 轨迹。无论它揭示什么,这份受控测量本身都是一个新的经验对象。
- **分析层(贡献取决于数据)**:给定轨迹后我们问:是否存在相变?是否有可拟合的边界 α*(N)?在留出设定上能否预测?跨家族/任务是否普适?崩溃是由假阳还是假阴驱动?

## 2. 相关工作及各自的局限

按攻关路线组织;每条都有一个**针对我们问题**的具体局限(方括号内为缺口)。

**(a) 带验证器的自训练 / 自我提升**:STaR(Zelikman 2022)、ReST(Gulcehre 2023)、ReST^EM「Beyond Human Data」(Singh 2023)、V-STaR(Hosseini 2024)、SPIN(Chen 2024)、Self-Rewarding LMs(Yuan 2024)。它们定义了我们采用的迭代循环。*局限:* 验证器要么是**近乎完美的真值检查器**(对答案 / 跑单测,α≈1),要么是**模型自己**的判断;没人把验证器准确度当变量扫,因此无法回答"验证器要多准"。

**(b) 奖励模型过优化与验证**:Gao、Schulman、Hilton(2023,「Scaling Laws for Reward Model Overoptimization」)用 gold/proxy 奖励模型给出了可控验证器,并拟合出关于 KL 的干净退化律;Feng、Kempe 等(2024,「Beyond Model Collapse」)用 (ϕ,ψ) 参数化不完美验证器,证明一次性相变阈值 p⋆ = 1/(1+ψ/ϕ)。*局限:* 二者都是**一次性**(单轮 / N→∞),因此看不到驱动迭代崩溃的跨轮复利,也没有模型规模轴。

**(c) 模型崩溃**:Shumailov 2024(Nature)、Gerstgrasser 2024(累积 vs 替换)、Dohmatob 2024(「Tale of Tails」,迭代修正的标度律)、Seddik 2024、Bertrand 2023(迭代再训练稳定性)。它们研究递归自训练动力学与崩溃。*局限:* 它们**没有验证器**(纯递归)或用模型自身似然;我们问题里的核心旋钮缺席。

**(d) 带噪声/不完美验证器的 RL(RLVR)**:Plesner 2026(「An Imperfect Verifier is Good Enough」,跨家族/规模扫验证器噪声 0–15%)、Rad 2026(「Rate or Fate?」,以 Youden's J = TPR−FPR 为旋钮,给出清晰 J=0 边界)、Bao 2025(FP/FN 噪声信道)。它们扫了验证器质量。*局限:* 它们在**在线策略 RL(GRPO 式)**里,不存在跨轮伪标签前馈;而自消费 SFT 的动力学——这一轮被接受的标签成为下一轮训练数据——是不同的。

**(e) 自我提升动力学 / solver-verifier gap**:Song 2024(「Mind the Gap」)、Huang 2024(「Sharpening Mechanism」,ICLR'25 Oral)、arXiv 2507.00075(solver-verifier-gap 动力学,拟合指数轨迹 + ε-收敛时间)、Yang 2025(2508.16456,闭式自我修正轨迹 Acc_t = Upp − αᵗ(Upp − Acc₀))。*局限:* 验证器是模型的**自我验证**(内部,不是外部/可扫);其律是关于自洽性或预训练 FLOPs,不是关于外部验证器准确度;轨迹**单调/饱和,没有崩溃支**。

**(f) 测试时扩展**:Brown 2024(「Large Language Monkeys」)、Snell 2024(测试时算力)。*局限:* **没有训练循环**——只在推理时。

**(g) 领域如何建模可控验证器**:Gao 的 gold/proxy(验证器质量=proxy-RM 容量);Feng/Kempe 的 (ϕ,ψ);Bao 的两率信道 `P(R̃=1|R*=0)=ρ₀, P(R̃=0|R*=1)=ρ₁`;Rad 的 Youden's J。领域标准是**两参数(假阳、假阴)**信道;单一对称准确度是最弱的变体。我们采用两参数形式。

**小结:** 每条路线都固定了其中一维:验证器完美(a)、一次性(b)、无验证器(c)、RL 非 SFT(d)、内部自我验证(e)、无训练(f)。**正中央那个组合是空的。**

## 3. 我们要桥接的 Gap

> 没有任何既有工作:把**外部验证器准确度作为连续可扫变量**(两参数 ρ₀, ρ₁),放进**迭代自消费 SFT**,跨**模型规模、家族、任务**,去问那条崩溃↔持续的边界是否可预测。

这就是我们要桥接的精确组合。它区别于一次性 RM 过优化(b)、无验证器的崩溃(c)、在线 RLVR(d)、以及内部自我验证(e)。

## 4. 我们的做法

**4.1 循环与协议(对标既有工作):** 采用 ReST^EM 循环:每轮当前模型对每个训练 prompt 生成 K 个候选解;验证器逐个接受/拒绝;在被接受的(prompt, 模型自生成解)对上做 LoRA 微调;重复。协议参数对标 ReST^EM/STaR/2507.00075:
- 训练 prompt:固定标准子集(≈2048,据 2507.00075)直至全量 GSM8K 训练集;
- 每题 K=16 采样,温度 1.0;每题最多保留 ≤10 个被接受解(ReST^EM);
- **主线动力学 = 每轮从 base 重启 + 替换(restart-from-base + replace)**(ReST^EM/STaR);continue/accumulate 作为消融;
- LoRA(r=16,attn+MLP)+ AdamW,lr 1e-4,每轮 2 epoch;
- 在**全量**留出测试集上评测,greedy pass@1(+ pass@k);
- **≥3 个 seed** 并报告方差(超过领域常见的 n=1)。

**4.2 可控验证器(旋钮):** 把验证器建模为相对真值的「接受/拒绝决策」上的两参数噪声信道——**ρ₀ = P(接受 | 解其实是错的)**(假阳;把错标签注入下一轮训练集 = 崩溃驱动)与 **ρ₁ = P(拒绝 | 解其实是对的)**(假阴;丢掉一个好样本)。真值答案(GSM8K 最终答案 / ARC 选项)**仅**用于定义验证器和评测,**绝不**作为训练目标。主扫:对称 ρ₀=ρ₁=ρ,验证器准确度 α=1−ρ ∈ {1.0,…,0.4}。**非对称切片**(只 ρ₀ vs 只 ρ₁)用来检验崩溃由假阳还是假阴驱动。

**4.3 各轴:** 验证器准确度 α(及 FP/FN 拆解)× 模型规模 N(Qwen2.5 0.5/1.5/3/7/14B)× 模型家族(Qwen、Llama-3.1-8B、Gemma-2-9B、OLMo-2-7B、DeepSeek-R1-Distill-Qwen-7B、Phi-3.5-mini)× 任务(GSM8K、ARC-Challenge);动力学消融(restart/continue × replace/accumulate)。

**4.4 基础设施:** vLLM 做高吞吐生成(每轮一次性子进程,干净释放 GPU)+ PEFT-LoRA 训练;每个(α, 模型, 任务, seed)单元产出完整的逐轮测试准确度轨迹。

**4.5 实验产出的东西(novelty,与结论无关):** 一份按(受控 ρ₀/ρ₁、规模、家族、任务、动力学)索引、在标准协议下的迭代自训练轨迹数据集——目前不存在的测量。其价值不依赖于分析结论。

## 5. 假设与预期结果

预注册的分析假设(数据后评估,不预设):
- **H1(存在性):** 存在一个临界验证器准确度 α*(N),分隔「持续提升」与「崩溃」。
- **H2(形式):** 有利可图轮数 / 边界是 (α, N) 的规则函数。
- **H3(可预测性):** 在除一个留出设定外的所有数据上拟合的模型,能预测该留出设定的崩/不崩。
- **H4(普适性):** 该边界跨家族/任务一致(相对于模型特异)。
- **H5(机制):** 崩溃由假阳(ρ₀)驱动,而非假阴(ρ₁)。

**上限(最好):** 一条干净、可拟合、**普适且可预测**的相界 α*(N) —— 一条 verifier-bounded 自我提升的可预测律,加上 FP/FN 拆解定位因果驱动。具备 field-relevant 价值(COLM/NeurIPS/ICLR 级)。

**下限(最差 / null):** 没有干净的普适律——异质、家族特异。即便如此,实验仍交付:**首份系统性测量**(验证器准确度对迭代自训练的影响)+ 一个对实践有用的可发表发现(「崩溃易感性不能仅凭规模/能力预测」)。非零(workshop / 经验研究级)。

**初步动机(受控,非 LLM 结果):** 在一个受控的表格化研究中(逻辑回归/MLP,合成 + 真实数据,无 LLM),我们观察到三区结构(崩溃 / 中间带 / 收敛)以及约 0.85 的留一格「崩/不崩」预测准确率。这为 LLM 测量提供动机;同样的结构与可预测性在 LLM 规模上是否成立,正是主实验要检验的。

## 6. 定位与诚实评估

- **实验层 novelty 是确定的**:受控的(验证器准确度 × 迭代 SFT × 规模/家族/任务)测量是新的;
- **分析层贡献是 conditional 的**:可预测的普适律(上限)vs 经验刻画 / 异质性结果(下限);
- **不是** Kaplan-2020 意义上的普适标度律;诚实定位 = 受控、可预测的刻画 + 机制(假阳驱动崩溃),并在真实小型 LLM 上验证;
- **风险**:周边领域(自我提升动力学、verifier-bounded RL)在 2024–2026 很活跃;速度 + 那个精确未被占据的组合是护城河。
